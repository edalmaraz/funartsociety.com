# Fun Art Society — public website design

**Date:** 2026-07-17 · **Approved:** Eric said "go end-to-end, do everything live" (design decisions delegated; this spec is the async review artifact).

## Purpose

Public face of Fun Art Society (funartsociety.com): umbrella brand + 9 product brands, each of which will sell digital products on Gumroad. The site must lend credibility to the storefronts, host a page per brand, and expose contact email. It is intentionally **static** — the FAS machinery (private repo `fun-art-society`, never published) can later update product listings by committing plain HTML/JSON to this repo.

## Non-goals

- No server-side code, no build toolchain, no JS framework.
- No mention of internal machinery (Paperclip, agents, nexus, myrig.fyi, pause states).
- No cart/checkout — Gumroad handles commerce (embed/overlay JS later).

## Approaches considered

1. **Single-page site** — simplest, but 9 brands in one scroll leaves no room for storefront pages to grow.
2. **Hand-rolled static multi-page with a data-driven generator** ← CHOSEN. `brands.json` + `gen_brands.py` stamp 9 uniform brand pages; index is hand-written. Zero toolchain, deterministic, future automation edits data not markup.
3. **Jekyll/11ty** — DRY templating but adds a build system for marginal gain at 11 pages.

## Architecture

```
index.html                  hero, brand grid (Coppergate featured), about, contact
brands/<key>/index.html     ×9, generated by gen_brands.py from brands.json
assets/site.css             single design system
assets/favicon.svg
404.html, robots.txt, .nojekyll
brands.json                 source of truth for brand pages
gen_brands.py               deterministic page generator (re-runnable)
verify_site.py              static QA gate (see below)
docs/superpowers/specs/     this spec
```

## Design language

Playful-but-credible collective. System font stack; one `--accent` custom property per brand page; light + dark via `prefers-color-scheme`; generous whitespace; emoji glyphs as brand marks (zero asset pipeline). Contact: hello@funartsociety.com (only after email routing is live; until then the contact section says email is "opening alongside the shops" — updated same session if routing completes).

## QA gate (`verify_site.py`, written before the site)

- Every `.html` parses; has `<title>`, exactly one `<h1>`, `meta description`, `lang`, viewport.
- Every internal href/src resolves to a real file; every brand in brands.json has a page; index links all 9.
- Forbidden strings (case-insensitive) nowhere in deployable files: `myrig`, `paperclip`, `nexus`, `authelia`, `agent`, `paused`, `localhost`, `127.0.0.1`, `conductor`, `warden`, `heartbeat`, `edalmaraz@` (noreply ok? no — no emails except @funartsociety.com).
- `CNAME` file, when present, contains exactly `funartsociety.com`.
